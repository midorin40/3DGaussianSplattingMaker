# GaussianSplattingMaker 要件定義

## 1. 背景

GaussianSplattingMaker は、ローカル環境にある COLMAP / Gaussian Splatting 実行系を、Web UI から扱うためのアプリです。

本プロジェクトでは、厳密な3D再構成だけを目的とせず、`.splat` データを生成してローカル表示や配布に使えることを重視します。

## 2. 目的

- 画像または動画から `.splat` データ生成フローを実行できること
- ローカル実行スタックを Web UI から扱えること
- 単一画像でも表示用途向けの `.splat` データ生成を試せること
- 実行状態、ログ、履歴をアプリ内で管理できること

## 3. 対象ユーザー

- ローカルで Gaussian Splatting を扱いたいユーザー
- 画像や動画から `.splat` を作りたい制作者
- 単一画像を表示用途向けに `.splat` 化したいユーザー
- 生成ジョブをブラウザから管理したい開発者

## 4. 必須要件

### 4.1 プロジェクト管理

- プロジェクトを作成できること
- プロジェクトに名前、モード、被写体種別、品質、背景処理、学習ステップ、目標フレーム数、メモを持てること
- プロジェクト履歴を保存できること

### 4.2 入力対応

- 画像アップロードに対応すること
- 動画アップロードに対応すること
- 動画はフレーム抽出して学習入力へ変換できること

### 4.3 実行モード

- 単一画像モードに対応すること
- 複数画像モードに対応すること
- 単一画像モードでは擬似ビュー生成を行えること

### 4.4 学習実行

- ローカルの COLMAP 前処理を起動できること
- ローカルの Gaussian Splatting 学習を起動できること
- `point_cloud.ply` を `.splat` に変換できること

### 4.5 ジョブ管理

- ジョブ進行状況を表示できること
- 実行ログを確認できること
- 学習中断ができること
- 学習再開ができること
- 学習ステップ数を指定できること
- 動画用の目標フレーム数を指定できること

## 5. 非機能要件

- 追加依存を最小限にした軽量構成であること
- Windows ローカル環境で動作すること
- ローカルファイルベースで状態保存できること
- API とフロントエンドがシンプルで追いやすいこと

## 6. 現在の仕様前提

- Web アプリ本体は `C:\WebApp\GaussianSplattingMaker`
- 実行スタックは `C:\GaussianSplatting`
- サーバーは Node.js 単体で動作
- データは `data/*.json` とローカルファイルで保存

## 7. 現在の制約

- 単一画像モードは完全3D化ではなく、表示用寄りの擬似ビュー生成ベース
- 動画品質は抽出フレーム数、目標フレーム数、元動画品質に強く依存
- `.splat` が生成されても、ビューア側のフォーマット差異で正常表示されない可能性がある

## 8. 今後の優先要件

- 単一画像モードの擬似ビュー数増強
- 表示用途向け `.splat` 品質の安定化
- フロントエンド上での出力確認導線強化
